설계 메모

기능 1: 언어 칼럼

  • 배치: "상태"와 "변경 URL" 사이 (유통 경로 앞) → 사이트 속성 그룹(분류→상태→언어)이 연속되어 자연스러움
  • DB: sites 테이블에 language VARCHAR(50) DEFAULT 'unset' 칼럼 추가
  • DB: site_languages 테이블 신규 생성 (유통 경로와 동일한 패턴)
  • API: GET /api/site-languages, POST /api/site-languages, PATCH /api/site-status/:domain/language
  • 통계 연동: GET /api/stats/by-domain에서 language 필드 JOIN 추가
  • 활동 이력: 너비를 기존 약 360px → 180px로 축소 (50%)

기능 2: 대기 중 URL 모달

  • API: GET /api/report-tracking/pending-summary → 모든 세션에서 report_status = '대기 중'인 URL 조회
  • 응답: { id, url, domain, report_id, created_at, report_status, reason }
  • 정렬: 신고일(created_at), 신고 ID(report_id) 오름차순/내림차순 (프론트엔드 정렬)
  • 상태 변경: 기존 PUT /api/report-tracking/:id/status 재사용 → 인라인 저장
  • 사유 변경: 기존 PUT /api/report-tracking/:id/reason 재사용
  • 리프레시 없음: 상태/사유 변경 시 로컬 state만 업데이트, 모달 닫을 때 부모 데이터 새로고침
